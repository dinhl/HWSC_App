Overview

This overview will give you some idea of what data is available, how to find what you need, and what you will see when you make a request to us.

The easiest way to see what data is included is to explore the data. You can build complex queries quickly and browse the results.

Please ensure that you read the guide for polling applications below before starting.

Results are returned in JSON format.

Endpoints

We provide several endpoints to retrieve different items:

  • Documents
  • Document Categories
  • Authors
  • Regions
  • Document items

For each endpoint:

  • Results can be filtered using parameters
  • Response contains minimal detail by default but more data can be exposed using parameters

Documents

The documents endpoint (/query/metadata) returns all pieces of content in the metadata database. For example, let's see all the contents of our metadata database:

http://localhost:3009/query/metadata?all

Document Categories

Document attributes can be fetched. For example, let's see all the keys of our metadata database:

http://localhost:3009/query/metadata?categories

Authors

All available authors and publishers of the data can be retrieved. For example, let's see all of our publishers in the database:

http://localhost:3009/query/metadata?uniqNames

Regions

All available regions of the data can be retrieved. For example, let's see all the regions of our data in the database:

http://localhost:3009/query/metadata?uniqRegions

Document Items

Querying specific documents can be filtered.

Parameters:

  • pi
  • firstName
  • region
  • date
  • fileType
  • instrument
  • samplingRate
  • dateSort

http://localhost:3009/query/metadata?param1=arg1,arg2&param2=arg1&paramN=argN

For example, let's see all the documents published by Seger and Faraon:

http://localhost:3009/query/metadata?pi=Seger,Faraon

Sort by date:

http://localhost:3009/query/metadata?all&dateSort=new

http://localhost:3009/query/metadata?all&dateSort=old
